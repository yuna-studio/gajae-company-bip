# 🦞 가재 컴퍼니 디자인 시스템 헌법 (RULES_DESIGN.md)

이 규칙은 가재 컴퍼니의 모든 디자인 프로젝트에 예외 없이 적용되는 영구 규칙입니다.

## 1. 디자인 시스템 준수 (Design System First)
- 모든 UI는 정의된 **디자인 시스템(DESIGN_SYSTEM.md)**을 기반으로 제작한다.
- 텍스트, 폰트, 컬러, 위젯(버튼, 다이얼로그, 바텀시트 등) 컴포넌트를 사전 정의한다.
- **개발 통제:** 개발자가 임의로 처음 보는 스타일을 사용하지 않도록 강력하게 통제한다.

## 2. 명세 전달 (Figma-Level Documentation)
- 디자인 시스템을 베이스로 피그마 수준의 상세 `.md` 명세서를 만들어 개발자/기획자와 소통한다.
- 필요 시 디자인 시스템 자체를 수정하여 일관성을 유지한다.

## 3. 디자인 QA (Visual Integrity)
- 개발자의 구현 결과물을 보고 실제 명세와 일치하는지 **디자인 QA**를 필수적으로 진행한다.
- 픽셀, 애니메이션 타이밍, 상태별 UI 변화가 의도대로 구현되었는지 검증한다.
